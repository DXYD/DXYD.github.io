Hexo+GitHub 搭建个人博客

准备

[^0]: 这里只在Windows10 64 位下演示

01安装Node.js

  • 打开官方网站安装所需版本的node.js

02安装Git

  • 打开官方网站安装所需版本的git

用git安装hexo

  • 新建一个文件夹

  • 右键选用git bush 打开

  • 输入node -v 通过查看版本判断是否已经安装好了node

  • 输入npm install -g hexo-cli安装hexo

  • 判断是否安装成功

  • 安装 hexo 完成后，请执行hexo init，hexo 将会在指定文件夹中新建所需要的文件

  • 输入npm install安装所需插件

  • 运行测试

     hexo clean #用来清理缓存文件
 hexo g     #生成文件
 hexo s     #运行本地服务器
 hexo d     #上传到服务器
    • 我们运行hexo g 、hexo s，成功后打开浏览器并且输入localhost:4000即可看到hexo的界面

部署到云端

01在github上新建仓库

  • 仓库名为: .github.io
  • 选择public

02将github page与hexo联系起来

  • 设置git的user name 和email

    $ git config --global user.name "xxx"
---
$ git config --global user.email "xxxxx@xxx.com"

  • 生成密钥文件

    ssh-keygen -t rsa -C "你的GitHub注册邮箱"

  • 然后直接三个回车即可，默认不需要设置密码

  • 找到生成的.ssh的文件夹中的id_rsa.pub密钥，将内容全部复制

  • 打开GitHub_Settings_keys 页面，新建new SSH Key

  • Title为标题，任意填即可，将刚刚复制的id_rsa.pub内容粘贴进去

  • 在Git Bash中检测GitHub公钥设置是否成功，输入ssh git@github.com

  • 打开站点的配置文件_config.yml，翻到最后修改为：

    deploy:

    ​ type: git

    ​ repo: ‘https://github.com/xxxx/xxxx.github.io.git'

    ​ branch: master

  • 安装Git部署插件，输入命令：

    npm install hexo-deployer-git --save

hexo d

我们分别输入三条命令：

hexo clean 
hexo g 
hexo d

hexo d 是部署网站命令，d是deploy的缩写

此时你就可通过 xxx.github.io 对你的博客进行访问了

最后

ssh：因为GitHub要求每次推送代码都是合法用户，所以每次推送都需要输入账号密码验证推送用户是否是合法用户，为了省去每次输入密码的步骤，采用了ssh，当然你也可以再每次推送时输入一遍账号和密码而跳过ssh这一步
